# Контент-план: статьи под боль клиента (SEO/AEO)

Главный канал роста по косвенным запросам. Люди и нейросети ищут НЕ «аудит сайта»,
а свою боль своими словами. Каждая статья ловит один такой запрос: answer-first
ответ на боль + мягкая ссылка на бесплатный разбор.

**Принципы (по docs/ai-seo-guide.md + seo-audit-guide.md):**
- Answer-first: первый абзац — само-достаточный ответ на запрос, с брендом, цитируемый.
- Заголовок статьи = формулировка боли дословно (как ищут).
- Структура: H1 (боль) → короткий ответ → H2 по подтемам → списки/конкретика → CTA.
- Каждая статья линкуется на главную (запустить разбор) и на 2-3 соседние статьи.
- FAQPage/Article schema на каждой.
- Без воды и AI-слопа, живым языком (docs/VOICE.md). Без фейк-цифр.
- Публиковать также на vc.ru / Habr (для Яндекс Нейро — он любит эти площадки).

## Формат каждой статьи (шаблон)

1. **H1** — боль дословно.
2. **Ответ-абзац** (2-3 предложения): прямой ответ + «проверить за пару минут можно на Razbor».
3. **H2: Частые причины** — список с конкретикой.
4. **H2: Как проверить у себя** — по шагам, ссылка на разбор.
5. **H2: Что чинить первым** — приоритет.
6. **CTA** — «Проверить свой сайт бесплатно».
7. **FAQ-блок** (2-3 смежных вопроса) + FAQPage schema.
