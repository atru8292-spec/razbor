# ТЗ: «Покажу, как бы я переделала первый экран» — демо-переделка хиро

Новая функция Razbor. После разбора — кнопка, по которой AI берёт первый экран
сайта, переделывает его по всем правилам (думает как дизайнер), показывает живой
превью и объясняет что и зачем поменял. Дразнит ценностью → ведёт к продаже
редизайна всего сайта.

---

## Зачем

Сейчас Razbor показывает ПРОБЛЕМЫ. Эта функция показывает РЕШЕНИЕ на примере
одного блока. Клиент своими глазами видит «переделанный вариант реально лучше» —
снимается возражение «а получится ли у неё». Не обещаем, а показываем. Усиливает
воронку к редизайну.

Грань: переделываем ТОЛЬКО первый экран (демо-дразнилка). Весь сайт — за деньги.
Демо должно быть достаточно хорошим чтобы захотелось, но это один блок, не всё.

---

## Поток (UX)

1. Клиент прочитал разбор. После блока «Первый экран» (или в конце выдачи) —
   кнопка «✨ Показать, как бы я переделала первый экран».
2. По клику (НЕ автоматом — экономим токены): генерация переделки.
3. Показывается:
   - Живой превью нового первого экрана (HTML/CSS, рендерится как настоящий).
   - Рядом/под ним — объяснения «что поменяла и почему» (по пунктам).
   - Внизу CTA: «Так можно переделать весь сайт. Обсудим? → @arinashrr».
4. Результат кэшируется (повторно не генерим, пока не нажал «переделать заново»).

---

## Архитектура (экономия токенов — важно)

КЛЮЧЕВОЕ: не анализировать сайт заново. Razbor УЖЕ проанализировал первый экран
в аудите — у нас есть находки по хиро, текст, скрин. Переиспользуем это.

```
[Кнопка «переделать первый экран»]
   ↓ берём из УЖЕ готового аудита:
   - находки по первому экрану (что не так)
   - текущий текст/структуру хиро
   - скрин первого экрана (если есть в аудите)
   - тип сайта, нишу, гео (уже определены)
   ↓ один вызов AI с жёстким промптом-дизайнером
[OpenAI gpt-5.4] → возвращает:
   - готовый HTML/CSS нового хиро (по дизайн-правилам)
   - список «что поменяла и почему» (объяснения)
   ↓
[Кэш в events] (admin_hero_redesign или поле в audit)
   ↓
[Показ в выдаче] — превью (iframe sandbox) + объяснения + CTA
```

Экономия:
- Генерим по кнопке, не в каждом аудите.
- Переиспользуем анализ из аудита (не платим за повторный разбор).
- Кэшируем (один раз сгенерили — показываем сохранённое).
- Модель gpt-5.4 (не mini — нужно качество дизайна), но один вызов на клик.
- Показывать стоимость в админке (как у AI-маркетолога).

Безопасность превью: HTML рендерить в sandboxed iframe (без скриптов клиента),
чтобы чужой контент не выполнил ничего. Только разметка + инлайн-стили.

---

## ПРОМПТ-ДИЗАЙНЕР (сердце функции — зашить дословно)

SYSTEM:
Ты сильный веб-дизайнер и CRO-эксперт. Тебе дают первый экран сайта и находки
аудита по нему. Задача — переделать первый экран так, чтобы он лучше продавал и
приводил к заявке, и объяснить каждое решение простым языком для владельца.

КАК ДУМАТЬ (по порядку):
1. Сначала пойми: что за бизнес, кто клиент, какое целевое действие.
2. Посмотри находки аудита — что конкретно не так с текущим первым экраном.
3. Переделай, закрывая эти проблемы, по правилам ниже.
4. Объясни каждое ключевое изменение: что было плохо → что сделал → почему лучше.

ПРАВИЛА ХОРОШЕГО ПЕРВОГО ЭКРАНА (делать так):
- Заголовок про ВЫГОДУ клиента и результат, не «мы команда профессионалов».
  Понятно за 5 секунд: что это, кому, какая польза.
- Один понятный главный CTA, кнопка говорит результат («Получить расчёт»),
  не «Отправить».
- Подзаголовок объясняет суть/механизм коротко.
- Видно доверие сразу если уместно (цифры, факты — без выдумок).
- Чистая визуальная иерархия: главное крупно, путь взгляда ясный.
- Мобайл: один экран, ничего не съезжает, кнопка видна без прокрутки.

ЗАПРЕТЫ (НЕ делать — это AI-слоп):
- НЕ фиолетово-синий градиент (#a855f7→#3b82f6) — главный маркер слопа.
- НЕ Inter/Roboto/Arial. Шрифты с характером (но web-safe или Google Fonts
  ссылкой, чтоб превью отрендерилось).
- НЕ три одинаковые карточки, не центрировать всё подряд.
- НЕ эмодзи как иконки, не ракеты, не блобы, не generic-тени.
- НЕ «Welcome to», «Empowering your journey», воду и клише.
- НЕ выдумывать факты/цифры/отзывы которых нет у клиента.

УЧИТЫВАЙ КОНТЕКСТ:
- Сохрани суть бизнеса и реальные данные клиента (название, услуги, контакты).
- Не выдумывай предложения которых нет. Улучшай подачу того что есть.
- Стиль адекватный нише (стоматология ≠ глэмпинг ≠ магазин).

ФОРМАТ ОТВЕТА (строго):
Верни JSON:
{
  "html": "<секция первого экрана: семантичный HTML с инлайн-CSS или <style>,
           самодостаточный, рендерится в iframe, адаптивный, по правилам выше>",
  "changes": [
    {"было": "...", "стало": "...", "почему": "простым языком зачем так лучше"},
    ... 3-5 ключевых изменений
  ]
}

Тон объяснений по docs/VOICE.md — живой эксперт, без жаргона, по делу.
Объяснения — это продажа экспертизы, делай их понятными и убедительными.

USER (подставлять из готового аудита):
Бизнес: {тип, ниша, гео}. Целевое действие: {action}.
Текущий первый экран: {текст/структура хиро}.
Находки аудита по первому экрану: {находки про hero из result}.
Переделай первый экран и объясни.

---

## Качество (критично)

Если переделка выйдет слопной/плохой — эффект обратный, клиент решит «фигню
предлагает». Поэтому:
- Промпт жёсткий по анти-слопу (выше) + сверка с docs/BUILD-GUIDE.md.
- На старте — хозяйка вычитывает первые 5-10 переделок вживую, калибрует промпт.
- Объяснения «почему» обязательны — они продают экспертизу, не только картинка.
- Лучше скромно но чисто, чем пёстро и слопно.

---

## Что НЕ делать
- НЕ переделывать весь сайт (только первый экран — это демо-дразнилка).
- НЕ автоматом в каждом аудите (только по кнопке — токены).
- НЕ выдумывать контент/услуги клиента.
- НЕ трогать логику аудита, JSON-схему разбора.

## Приёмка
- Кнопка «Показать как бы я переделала первый экран» после разбора.
- По клику — генерация из готового аудита (не повторный анализ), кэш, стоимость.
- Превью нового хиро рендерится (sandboxed iframe), выглядит чисто, не слоп.
- Объяснения «было/стало/почему» по-человечески, по VOICE.md.
- CTA к редизайну всего сайта внизу.
- Промпт сверяется с BUILD-GUIDE, анти-слоп держится.
- Первые переделки вычитаны хозяйкой, промпт откалиброван.

---

## Этап запуска
1. Сначала сама механика: кнопка → генерация → превью → объяснения. На 2-3 сайтах.
2. Хозяйка смотрит качество переделок, калибрует промпт (не слоп ли, по делу ли).
3. Потом — отшлифовать превью (адаптив, рамка-«вот ваш экран»), CTA.
4. Дальше по желанию — переделка не только хиро, а ещё блока (но осторожно с
   ценностью: много бесплатно = незачем платить за редизайн).
