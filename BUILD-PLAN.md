# План сборки — для Claude Code

Идём строго по фазам, по одной за раз. После каждой фазы — чекпойнт: проверила, закоммитила, потом `/clear` и следующая.
Каждую фазу начинаем в plan mode (Shift+Tab) — Claude Code распишет шаги, ты смотришь, подтверждаешь, и только потом она пишет код.

Промпты ниже копируешь как есть. Она сама прочитает `CLAUDE.md` и `docs/TZ.md`.

**Промпт:**

Воркер-роут выполняет пайплайн по шагам и пишет progress в Supabase после каждого. GET /api/audit/:id отдаёт статус и результат. Пока пайплайн = заглушка с фейковыми шагами и задержками. Сначала план.

**Чекпойнт:** POST возвращает id, GET показывает как progress растёт от pending до done. Async-скелет живой. Коммит.

## Шпаргалка по Claude Code
- `Shift+Tab` — переключить plan mode (планирует, не пишет код, пока не подтвердишь)
- `/clear` — очистить контекст между несвязанными задачами
- `/init` — если надо, чтобы она сама дописала CLAUDE.md под код
- проси «запусти build и закоммить когда зелёно» — выполнит сама
- застряла или ушла не туда — Esc прерывает, git откатывает
